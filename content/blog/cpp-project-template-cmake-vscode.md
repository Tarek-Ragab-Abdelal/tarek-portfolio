I originally built this template for personal use, then shared it because the same onboarding problem appears repeatedly:

- New developers struggle to set up a clean C++ workspace.
- Teams lose time rewriting the same project scaffolding.

## Template goals

The template was designed to be practical from day one:

- Clear folder structure.
- Straightforward CMake configuration.
- Easy VSCode integration.

## Why structure matters

A reliable template improves learning speed for students and reduces setup friction for professional teams. It also encourages better habits around modular code organization and build reproducibility.

## Key lesson

Project structure is part of engineering quality. Spending time on a good baseline pays off across every feature you build afterward.
